// INITIAL SETUP
Supabase 接続設定
初回のみ設定が必要です。supabase.com でプロジェクトを作成し、
Project Settings → API からURLとAnon Keyを取得してください。
▶ Supabase SQLセットアップ手順を見る
Supabase の SQL Editor で以下を実行してください： 
-- 1. users テーブル
create table if not exists users (
  id         bigserial primary key,
  name       text not null,
  company    text default '',
  email      text default '',
  login_id   text not null unique,
  pass       text not null,
  role       text default 'member',
  status     text default 'approved',
  is_first_login boolean default true,
  joined     text default ''
);

-- 2. articles テーブル
create table if not exists articles (
  id             bigserial primary key,
  title          text not null,
  category       text default 'その他',
  type           text default 'free',
  excerpt        text default '',
  body           text default '',
  date           text default '',
  emoji          text default '💡',
  show_reactions boolean default true,
  attachments    jsonb default '[]'::jsonb
);

-- 3. reactions テーブル
create table if not exists reactions (
  article_id bigint,
  user_id    bigint,
  type       text,
  primary key (article_id, user_id)
);

-- 4. Storage バケット (Storage タブで作成)
-- バケット名: henjinkai  / Public: ON

-- 5. RLS を無効化（シンプル運用の場合）
alter table users    disable row level security;
alter table articles disable row level security;
alter table reactions disable row level security;

-- 6. 管理者アカウント初期データ
insert into users (name, company, email, login_id, pass, role, status, is_first_login, joined)
values ('管理者', '変人会', 'admin@henjinkai.jp', 'admin', 'admin123', 'admin', 'approved', false, '2024-01-01')
on conflict (login_id) do nothing;
